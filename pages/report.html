Job Description

During the first four months of my internship at Electronic Arts (EA), I served as a C++ Software Engineer on the Client Team for the new FC24 game, working on both front-end and back-end features. My primary responsibility during this phase of my internship was centered around bug fixing within the Ultimate Team (UT) game mode. This task involved the identification, testing, and resolution of various bugs, each presenting unique challenges and valuable learning experiences. I worked extensively with C++, ActionScript, and XML in this capacity. As I became more immersed in the role of a Software Engineer, my responsibilities expanded to include redesigning aspects of the game, introducing new features and functionalities, and managing crash dump reports, which I found particularly intriguing. These reports represented errors in the codebase that led to game crashes during specific in-game actions. My role entailed identifying the source of the error and determining whether it originated within our team or elsewhere, necessitating appropriate communication and action. This experience greatly enhanced both my technical and interpersonal skills, as it involved a well-defined sequence of tasks. These experiences were like a big step forward in my learning as they provided insights into the company's workflow and the entire process of implementing changes into the mainstream game.

Most of the skills needed for the job were learned during onboarding. This included becoming familiar with company software like Perforce, Jira, Slack, and Frostbite Drone, as well as participating in stand-up meetings and understanding the bug ticketing process. However, one critical skill I could directly apply from my school background was C++ programming. In the first two years of my education, we extensively covered C and Object-Oriented Programming (OOP) concepts, which made it relatively easy to transition to a programming language I had previously learned. Consequently, I was able to quickly enhance my proficiency in C++. Moreover, I leveraged this knowledge to pick up a new programming language, ActionScript, which was used by the company and proved to be highly beneficial.

Job Description

During my 8-month internship at Electronic Arts (EA), I worked as a C++ Software Engineer on the Client Team for the new FC24 game. I was involved in both front-end and back-end features. For the first 6 months, my main task was bug fixing within the Ultimate Team (UT) game mode. I identified, tested, and resolved various bugs using C++, ActionScript, and XML. As I grew in my role, I redesigned game aspects, introduced new features, and managed crash dump reports. These reports highlighted codebase errors causing game crashes. My job was to find the error source and its origin, which improved my technical and interpersonal skills. This experience was a big step in my learning, giving me insights into the company’s workflow and how changes are implemented into the game.

Most of the skills needed for the job were learned during onboarding. This included getting familiar with company software like Perforce, Jira, Slack, and Frostbite Drone, participating in stand-up meetings, and understanding the bug ticketing process. However, one important skill I could apply from my school background was C++ programming. In the first two years of my education, we covered C and Object-Oriented Programming (OOP) concepts extensively, which made it easier to transition to a programming language I had previously learned. As a result, I was able to quickly improve my proficiency in C++. Moreover, I used this knowledge to learn a new programming language, ActionScript, which was used by the company and proved to be very useful.

For the last 2 months of the internship, we moved into pre-production and improvement staging tasks. So my job role changed from fixing bugs and game crashes to developing new scripts and features that can be implemented in the next game cycle to improve the game. Also, as a task I wanted to work on, I took my own initiative to tackle a challenge I faced during my internship which can benefit the team. This was to develop an automated script that can help new software engineers find screen code much faster in our large code base. During the FC 24 cycle, as a new SE intern on the team, I found it quite challenging to quickly adapt to the codebase and locate the relevant code for the current screen in order to resolve bugs. This led to a significant waste of time when trying to address these issues.

The main reason for this challenge is the sheer size of the codebase at EA, which requires a lot of time to fully understand the workflow. As a result, it makes it hard to grasp the concept of finding code in the early stages of onboarding. To solve this problem, a script is currently being developed, which will be a valuable tool for pinpointing the code responsible for specific screens as accurately as possible. This tool is expected to be very helpful for new SEs on the FUT Client Team, but it will also be available for use by experienced SEs who may find it useful in their work.
